Déroulement du projet

Le projet a débuté par l'initialisation d'un nouveau projet Django et la configuration de l'environnement virtuel. J'ai ensuite défini les modèles Région et Château dans le fichier models.py, avant de créer les vues et les formulaires correspondants dans views.py et forms.py. Les templates HTML ont été mis en place pour la page d'accueil, la liste des régions, la liste des châteaux et les pages de détails. Enfin, j'ai testé l'application en local et configuré l'interface d'administration Django pour la gestion rapide des données.

Présentation du sujet

Développement d’une interface de gestion du trafic aérien entre aéroports, permettant au gestionnaire de créer et administrer les entités suivantes : aéroports, pistes d’atterrissage, compagnies, types d’avion, avions et vols.

Schéma de données

  • Aéroports (id, nom, pays)
  • Pistes (numéro, aéroport, longueur)
  • Compagnies (id, nom, description, pays de rattachement)
  • Types d’avion (id, marque, modèle, description, images, longueur de piste nécessaire)
  • Avions (id, nom, compagnie, modèle)
  • Vols (id, avion, pilote, aéroport de départ, date et heure de départ, aéroport d’arrivée, date et heure d’arrivée)

Fonctionnalités principales

  • CRUD complet pour chaque entité (aéroports, pistes, compagnies, types d’avion, avions, vols).
  • Ajout interactif de nouveaux avions, pistes et vols.
  • Vérification automatique de la disponibilité d’une piste à l’heure d’arrivée (usage de 10 min par vol), proposition d’horaire ultérieur si indisponible.
  • Import en masse de vols depuis un fichier structuré (format décrit dans l’aide avant chargement).
  • Génération de fiches de vols au départ ou à destination d’un aéroport pour une période définie.

Import de données

Préparation de la base de données initiale avec un jeu de données : aéroports, pistes, compagnies, types d’avion et avions.

Page d’import : description du format attendu pour le fichier de vols (CSV ou JSON), champs requis et exemples.

Technologies employées

  • HTML5, CSS3, JavaScript
  • Django (Python)
  • Base de données relationnelle (MariaDB ou PostgreSQL)
  • Serveur web (Nginx + Gunicorn)

Contraintes et solutions

Contrainte de réservation des pistes : gestion des créneaux pour éviter les conflits. Implémentation de règles métier pour allouer ou proposer des plages horaires alternatives.

Validation du format de fichier d’import et gestion des erreurs (lignes incomplètes, données invalides).

Résultats attendus

  • Interface web intuitive pour gérer toutes les entités du trafic aérien.
  • Système robuste de réservation de pistes avec suggestion d’alternatives.
  • Import et export de données facilitant la maintenance.
  • Fiches de vols générées dynamiquement selon les requêtes.

Perspective pédagogique

Projet visant à maîtriser la conception d’un modèle de données complexe, la mise en place de règles métier et le développement d’interfaces CRUD avec contrôle de disponibilité et import de données.
